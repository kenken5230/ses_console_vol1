# 市場分析機能 MVP提案

## 方針

最初のMVPは、既存DBを読み取るだけで実現できる市場分析に限定する。DB変更、migration、既存画面変更、外部API、AI APIは使わない。

目的は、営業が毎日見て「今どこを攻めるか」を判断できる画面を作ること。成約予測モデルや高度なAIマッチングは、履歴データがそろってから進める。

## MVPに含める機能

### 1. 市場分析トップ画面

表示するもの:

- 案件数
- 要員数
- 注力案件数
- 単価未設定件数
- スキル未設定件数
- 地域未設定件数
- 要確認抽出件数
- 上位の営業注力セル

用途:

- 営業会議で市場状況を一目で確認する
- データ品質の穴を見つける

### 2. スキル別ランキング

指標:

- 案件数
- 募集人数合計
- 要員数
- 需給ギャップ
- 案件単価中央値
- 要員希望単価中央値
- 必須スキル需要
- 尚良スキル需要
- 注力案件比率

優先理由:

- 既存の `ProjectSkill` と `PersonSkill` で出せる
- ChatGPT側調査でも重要候補
- AIマッチング特徴量にも直結する

### 3. 単価帯別ランキング

単価帯:

- 〜50万円
- 50〜60万円
- 60〜70万円
- 70〜80万円
- 80万円〜
- 未設定

指標:

- 案件数
- 要員数
- 案件単価中央値
- 要員希望単価中央値
- 単価乖離
- 高単価かつ要員不足のセル

優先理由:

- 営業注力と価格交渉に使いやすい
- 単価妥当性判定の土台になる

### 4. 地域・勤務形態別ランキング

指標:

- 地域別案件数
- 地域別要員数
- 地域別単価中央値
- フルリモート / 一部リモート / 常駐の案件数
- 地域 x 勤務形態の需給ギャップ

優先理由:

- SES営業では地域とリモート条件が成約可否に強く影響する
- 案件側は `prefecture` と `remoteType` がある

注意:

- 要員側の勤務地・リモート希望は自由記述なので、MVPでは簡易正規化に留める。

### 5. 需給ギャップランキング

市場セルを縮約してランキングする。

```text
正規化スキル x 単価帯 x 地域 x 勤務形態 x 開始月 x 契約形態
```

表示する指標:

- 案件数
- 募集人数
- 要員数
- 需給ギャップ
- 案件単価中央値
- 要員希望単価中央値
- データ品質警告数

優先理由:

- 今回の市場分析の中核
- 将来のAIマッチング特徴量に転用できる

### 6. 営業注力候補一覧

候補セルごとに、営業優先度スコアを出す。

```text
営業優先度スコア =
  募集人数スコア
  + 案件数スコア
  + 高単価スコア
  + 需給ギャップスコア
  + 注力案件スコア
  + 開始月近さスコア
  - データ不足ペナルティ
```

最初は説明可能なルールベースにする。

表示例:

| ランク | 市場セル | 理由 | 推奨アクション |
|---|---|---|---|
| 1 | Java x 70〜80万円 x 東京 x 常駐 x 来月 | 案件多、要員不足、開始月近い | Java要員の掘り起こし |
| 2 | AWS x 80万円〜 x フルリモート | 高単価、要員不足 | インフラ/BP営業強化 |

### 7. データ不足アラート

アラート対象:

- 案件スキルなし
- 案件単価なし
- 案件地域なし
- 案件開始月なし
- 要員スキルなし
- 要員希望単価なし
- 要員稼働開始日なし
- `remoteType = UNKNOWN`
- `ExtractionResult.reviewStatus = NEEDS_REVIEW`
- `reviewReasons` あり

優先理由:

- 分析精度を上げる即効性がある
- AIマッチング前の品質管理にも使える

### 8. 注力案件との関連分析

表示するもの:

- 注力案件のスキル分布
- 注力案件の単価帯分布
- 注力案件の地域分布
- 注力案件の契約形態分布
- 注力案件に対して要員供給が足りないセル

優先理由:

- 現場の注力判断とデータ上の需給ギャップを比較できる

## MVPから外す機能

| 機能 | 後回し理由 |
|---|---|
| AIによる自動提案 | 既存マッチング結果履歴と教師データが不足 |
| 成約率予測 | 成約・失注・面談結果の履歴が不足 |
| 粗利ランキング | 契約単価、仕入単価、粗利が不足 |
| 継続しやすい案件条件分析 | 契約開始・終了・延長・終了理由が不足 |
| 顧客業界別分析 | 顧客業界カラムが不足 |
| 外部市場データ連携 | 今回は外部API禁止。内部データ優先 |
| migrationを伴う特徴量保存 | 今回はDB変更禁止 |
| `/matches` 画面改修 | 今回のテーマ外。既存機能への影響を避ける |
| CSVインポート改修 | 今回のテーマ外 |
| メール生成・送信改修 | 今回のテーマ外 |

## MVPで使う既存データ

| データ | 使用目的 |
|---|---|
| `Project.status` | 募集中案件の絞り込み |
| `Project.isFocus` | 注力案件分析 |
| `ProjectSkill` | スキル別需要 |
| `ProjectCondition.unitPrice*` | 案件単価帯 |
| `ProjectCondition.upperAmount*` | 上位金額ベースの単価 |
| `ProjectCondition.recruitingCount` | 需要量 |
| `ProjectCondition.prefecture` | 地域 |
| `ProjectCondition.remoteType` | 勤務形態 |
| `ProjectCondition.startMonth` | 開始月 |
| `ProjectCondition.contractType` | 契約形態 |
| `ProjectCompanyRole` | 商流 |
| `Person.status` | 稼働可能要員の絞り込み |
| `PersonSkill` | スキル別供給 |
| `Person.desiredUnitPrice` | 要員希望単価 |
| `Person.availableFrom` | 稼働開始 |
| `Person.preferredLocation` | 希望勤務地 |
| `Person.remotePreference` | リモート希望 |
| `ExtractionResult` | データ品質 |

## MVPの画面イメージ

1. サマリーカード
   - 案件数、要員数、注力案件数、要確認数、単価未設定数
2. ランキングタブ
   - スキル
   - 単価帯
   - 地域/勤務形態
   - 市場セル
   - データ品質
3. フィルター
   - 期間
   - 開始月
   - スキル
   - 地域
   - 単価帯
   - 勤務形態
   - 契約形態
   - 注力案件のみ
4. 詳細ドリルダウン
   - セルを選ぶと該当案件・該当要員の件数と代表例を表示

## 成功条件

- 営業が「今月攻めるスキル・地域・単価帯」を3分以内に把握できる
- データ不足項目が明確になり、入力改善につながる
- 既存DBだけで表示できる
- 既存の案件管理、要員管理、メール、CSV、マッチング画面に影響しない
- 将来のAIマッチング特徴量に流用できる指標名・集計単位になっている
